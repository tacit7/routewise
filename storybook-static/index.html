Component Documentation

Complete technical documentation of Google OAuth authentication flow implementation in RouteWise.

Architecture Overview

Frontend (React + Vite)

  • • Port 3001 (development)
  • • Vite proxy configuration
  • • AuthContext for state management
  • • JWT token handling
  • • Responsive UI components

Backend (Phoenix/Elixir)

  • • Port 4001 (API server)
  • • Ueberauth + Google Strategy
  • • JWT token generation
  • • Session management
  • • Database user storage

Authentication Flow Steps

1

User Clicks Sign Up

User clicks the Sign Up button on the home page, triggering OAuth initiation.

window.location.href = '/auth/google'
2

Vite Proxy Forward

Vite development server proxies the auth request to Phoenix backend.

localhost:3001/auth/google → localhost:4001/auth/google
3

Phoenix Router Match

Phoenix router matches the OAuth route and applies browser pipeline with Ueberauth.

GET /auth/google → AuthController.request
4

Google OAuth Redirect

Ueberauth redirects user to Google OAuth consent screen.

→ accounts.google.com/oauth/authorize
5

User Authorization

User authorizes the application on Google's consent screen.

6

Google Callback

Google redirects back to Phoenix with authorization code.

GET /auth/google/callback

Vite Proxy Configuration

// vite.config.ts
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:4001',
      changeOrigin: true,
    },
    '/auth': {
      target: 'http://localhost:4001',
      changeOrigin: true,
    },
  },
}

Phoenix Router Configuration

# router.ex
pipeline :auth_browser do
  plug :accepts, ["html"]
  plug :fetch_session
  plug Ueberauth
end

scope "/auth" do
  pipe_through :auth_browser
  get "/:provider", AuthController, :request
  get "/:provider/callback", AuthController, :google_callback
end

API Endpoints

GET /auth/google

Initiates Google OAuth flow

GET /auth/google/callback

Handles Google OAuth callback

GET /api/auth/me

Gets current user (requires authentication)
